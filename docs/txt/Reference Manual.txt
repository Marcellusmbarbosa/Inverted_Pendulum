1 Introduction
The software for the Digital Pendulum system consists of:
• A Real-Time Kernel (RTK),
• Communication functions,
• Pendulum Toolboxes,
• External Interface,
• Windows NT kernel-mode device driver (for Windows NT only).
The Real-Time Kernel supervises a set of  real-time tasks creating a feedback control
structure (Fig. 1.1). The RTK provides a mechanism for  real-time measurements and
control of the digital -pendulum system in the MS-WINDOWS environment. It has the
form of a dynamic linked library (DLL) and is written in closed form. No modifications are
necessary. RTK contains all functions which are required for control and data collection in
real-time.
There are two different RTKs for the different versions of MATLAB and different
operating systems, which must be selected during software installation time.
1)  RTK for MATLAB ver. 5.x and Windows 95,
2)  RTK for MATLAB ver. 5.x and Windows NT operating system.
Control algorithms are embedded in the real-time kernel and their selection and tuning of
parameters is done by means of the communication interface from the MATLAB
environment. The communication interface is also used for configuration of real-time
kernel parameters (sampling period, excitation source etc.).
Specialised communication functions are responsible for communication between RTK
and MATLAB environment These functions are described in this manual.
The Pendulum Toolbox consists of a collection of M-functions (among them are S-
functions as well), Simulink models and C-code DLL-files that extend the MATLAB
environment in order to solve pendulum modelling/design/control problems. A cyclic
memory buffer is created in order to  enable process data flow between the real-time
kernel and toolbox functions (MATLAB environment). The Pendulum Toolbox, using
MATLAB matrix functions and Simulink utilities, provides the user with functions specific
for the real-time control of the digital pendulum system.

The toolbox is designed as an open system ( as compared to the RTK which is a closed
system). This approach makes the basic functions of the toolbox available to the user and
enables the user to create a tailored made system, add to it new control algorithms and
subsequently customise it further to satisfy the  requirements.
However, there are also demonstration M-files for typical pendulum control problems
available in the toolbox. And these are described in the Digital Pendulum System
GETTING STARTED Manual (33-005-1M5)
The External Interface for the digital pendulum system offers a way for the more
experienced user to extend the capabilities of the software. Although the RTK and
MATLAB interface create a complete, self contained environment for real-time
experiments, its applicability is limited to a fixed number of embedded controllers. The
interface provides a way for the user to design and add new controllers to the RTK and
implement them in real-time.
The kernel-mode device driver which gives the RTK access to the I/O address space is
required in the Windows NT environment.
The following pages contain a description of the Communication Toolbox functions. There
are 21 Communication Toolbox functions which are listed in alphabetic order and which
are divided into the four following categories according to their specific assigned roles.
•
hardware: 
GetBaseAddress, SetBaseAddress, ResetEncoder
•
data acquisition: 
GetNoOfSamples, ResetTime, GetSampleTime, 
SetSampleTime, StartAcq, GetHistory
•
software: 
LoadLibrary, UnloadLibrary
•
control: 
all ten remaining functions.

2 Description of the Toolbox Functions
All functions responsible for communication between the Real-Time Kernel (RTK) and
MATLAB environment are in the following form:
ReturnValue = pd_call( FunctionName, [ Argument ] )
where:
ReturnValue - value returned by the function,
pd_call - name of the DLL library responsible for the communication,
FunctionName - name of the desired operation, string format,
Argument - argument passed to the pd_call function (optional).

2.1 GetAlgNo
open loop excitation
embedded algorithms
rule-based  swing up
internal excitation source
Purpose:  
Get number of the currently active control algorithm.
Synopsis:  
AlgNo = pd_call( 'GetAlgNo' )
Description:
The function returns the number of the algorithm currently active in
RTK.
See:
SetAlgNo
Example:
The following function displays the description of the currently active
control algorithm.
function dspdsr
AlgNo = pd_call( 'GetAlgNo' );
if AlgNo == 0
  disp( 'Control set to zero' );
elseif AlgNo == 1
  disp( 'Open-loop control' );
elseif AlgNo == 2
  disp( 'Rule-based controller' );
elseif AlgNo == 3
  disp( 'LQ crane mode controller' );
elseif AlgNo == 4
  disp( 'Fuzzy controller' );
elseif AlgNo == 5
  disp( 'PID crane mode controller' );
elseif AlgNo == 99
  disp( 'External controller' );
else
  disp( '!!! Unknown controller !!!' );
end

sequence will never terminate:
while( pd_call( 'GetNoOfSamples' ) < 1200 ) ;
end;

2.14 LoadLibrary
Purpose:  
Load RTK to memory.
Synopsis:  
Count = pd_call( 'LoadLibrary' )
Description: 
The function loads the pd_dll.dll library module. If the pd_dll.dll
library was loaded previously LoadLibrary function only increments
(increases by one) the reference counter of the module. The library
can not be loaded to the memory again. If the module has not been
loaded previously, the function loads it from the pd_dll.dll file and
sets the reference counter to 1. The LoadLibrary increments the
reference counter each time MATLAB calls this function. If you want
to finish using the module, you should use the UnloadLibrary
function to decrement (decrease by one) the reference counter. The
pd_dll.dll library is removed automatically from the memory when the
module reference counter reaches zero.
First call to any pd_call function loads the pd_dll.dll module into
memory automatically.
The function returns the module reference counter.
MATLAB 5
In the RTK version for MATLAB ver. 5 the pd_dll.dll file does not exist. All functions are
included in the pd_call.dll module. This module does not contain the reference counter.
The first call of the pd_call function loads the module into memory. The call to the
UnloadLibrary function removes the module from memory.

3 Information Flow between Simulink Models and the Real-Time Kernel
This section describes an example of an S-function exchanging information between the
Simulink model and the Real-Time Kernel. S-functions were developed to give Simulink
the ability to construct generic simulation blocks, to handle different roles in one standard
form, such as continuous simulation, discrete simulation, systems embedded within
systems, and so on. In our applications special S-functions perform information exchange
between Simulink models and the Real-Time Kernel.
This section explains the use of Simulink models as the graphical front-end of the real-
time control program. It is important to realise that user-defined  S-functions are at the
heart of how your control experiments are performed. Inside a proper  S-function you can
select a real-time controller, set its parameters and make process data available within
the MATLAB environment. The masking mechanism, available in Simulink, allows you to
define a block in terms of its dialogue box, its icon and initialisation commands.
It is assumed that the reader is familiar with S-function formats but, if more details are
required, refer to  "Simulink. Dynamic System Simulation Software. User's Guide".
MATLAB 5
The principles of data flow between Simulink models and the Real-Time Kernel differ in
MATLAB version 4 and MATLAB version 5. There are different structures of S-functions
as well.

4 Guide to the Graphical User Interface (Examples)
The graphical user interface (GUI) is a set of  user-configurable graphical objects , such
as menus, buttons, lists and fields. MATLAB provides functions that enable a user to add
features of a GUI to the MATLAB application. This section describes these features and
shows how to use them for tuning or changing the form of the application.
MATLAB GUI elements fall into two classes:
• Controls
• Menus
In this manual only the Control class of GUI elements will be presented using example
applications. Controls are graphics objects such that, when manipulated with the mouse
cause a desired action. A control can perform an action with immediate results or can
change settings that modify future actions.
MATLAB 5
The way of creating GUI elements illustrated in this section is mainly used in the MATLAB
version 4 environment. In MATLAB version 5 the guide tool should be used.

4.1 TYPES OF CONTROLS
• 
Push Buttons are small screen objects usually labelled with text. Clicking the
mouse button on a push button causes MATLAB to perform a defined action.
• 
Check Boxes let the user select one or more options among  available
alternatives. Check boxes act like toggle switches, indicating the state of “on” or
“off”. The state is “on” if the box is checked.
• 
Radio Buttons let the user choose among mutually exclusive alternatives. Like
check boxes, radio buttons act as toggle switches, indicating the state of either “on”
or “off”.
• 
Sliders let the user choose a value within a range of values. Sliders are devices
which display their values graphically. The user can change the value by moving
the indicator.
• 
Pop-up Menus let the user choose an item from a list. Pop-up menus are used for
choosing a value from a list of values.
• 
Static Text displays a single line of textual information. It is typically used to label a
group of other controls, provide direction for the user, or indicate values associated
with a slider.
• 
Editable Text let the user enter a string value to be used by the application. The
value can consist of one or more lines of text. The user can accept, edit, delete, or
replace an editable text value.
• 
Frames let the user enclose regions of the figure window in a frame. You can then
place other controls within the frame to group them visibly.
To create controls use the uicontrol function:
h=uicontrol(hfig, ‘PropertyName’,’PropertyValue’,...)
hfig - specifies the figure handler, where a control is defined. If you omit the figure
handler, the control appears in the current figure window. If there is no figure window,
MATLAB creates a new one.
PropertyName/PropertyValue are pairs describing controls. When specifying properties,
you need to enter only enough characters to uniquely identify the property. Examples  of
the properties are given below.
